# Quick Start برای سازمان‌های ایرانی در حکمرانی داده

> وضعیت: در حال توسعه (Stub)

این راهنما در نسخه‌های بعدی یک مسیر گام‌به‌گام برای سازمان‌های ایرانی ارائه خواهد داد؛ از:

1. شناسایی دامنه‌های اصلی داده
2. تعیین نقش‌ها و مسئولیت‌ها
3. استفاده از Templateها (فرم درخواست داده، موافقت‌نامه اشتراک داده، ماتریس دسترسی)
4. انتخاب یک ابزار اوپن‌سورس مناسب (مثلاً OpenMetadata یا DataHub)

تا ساخت یک برنامه‌ی ساده و قابل اجرا برای حکمرانی داده.
