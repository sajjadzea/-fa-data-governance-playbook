# از کجا شروع کنیم؟ (براساس نقش)

## برای سازمان‌ها و مدیران (سیاست‌گذار، مدیر داده، مدیر خدمت)

1. مرور [README.md](../README.md) برای آشنایی سریع با هدف پلی‌بوک و اجزای اصلی.
2. مطالعه‌ی [از داده تا ارزش](data-to-value.md) برای دیدن مسیر تبدیل داده به تصمیم و خدمت بهتر.
3. مرور تمپلیت‌های کلیدی در [`templates/`](../templates/) مانند [فرم درخواست داده](../templates/data-request-form.md) و [ماتریس دسترسی](../templates/access-matrix.md) برای تعریف مسئولیت و دسترسی.
4. انتخاب یک دامنه کوچک داده و اجرای پایلوت با استفاده از [دفترچه آزمایش](../templates/experiment-log.md) برای ثبت فرضیه، اقدام و نتیجه.
5. بررسی [راهنمای ابزارها](tools/) و انتخاب یک ابزار اوپن‌سورس سبک برای شروع (مثلاً OpenMetadata یا DataHub) و مستندسازی اتصال اولیه.
6. بازبینی نتایج پایلوت و به‌روزرسانی سیاست‌ها، فرم‌ها و نقش‌ها براساس بازخورد کاربران و تیم‌ها.

## برای پژوهشگران و دانشجویان

1. مرور سناریوهای واقعی در [`scenarios/`](../scenarios/) برای شناخت موقعیت‌های متداول درخواست و اشتراک داده.
2. مطالعه‌ی [استفاده‌ی مسئولانه و اخلاقی از داده](ethical-data-use.md) برای درک اصول حریم خصوصی و شفافیت در پژوهش.
3. استفاده از [فرم درخواست داده](../templates/data-request-form.md) برای مستندسازی نیاز پژوهشی و محدوده داده.
4. در صورت کار با داده شخصی، طراحی فرایند ناشناس‌سازی و مجوزها براساس [ماتریس دسترسی](../templates/access-matrix.md).
5. ثبت درس‌آموخته‌ها و خروجی‌ها در [دفترچه آزمایش](../templates/experiment-log.md) تا بازنگری پژوهش آسان باشد.

## برای توسعه‌دهندگان، تحلیل‌گران و فعالان فنی داده

1. مطالعه‌ی بخش ابزارها در [`docs/tools`](tools/) برای آشنایی با گزینه‌های پیاده‌سازی سبک در محیط تست.
2. بررسی [تمپلیت‌های دسترسی و درخواست داده](../templates/) و هم‌راستا کردن APIها، Pipelineها و داشبوردها با این الگوها.
3. طراحی یک پیاده‌سازی حداقلی (MVP) از حکمرانی داده روی یک پروژه کوچک؛ شامل تعریف نقش‌ها، ثبت متادیتا و فعال‌سازی لاگ دسترسی.
4. مستندسازی نتیجه و پیشنهاد گسترش در سطح سازمان در [دفترچه آزمایش](../templates/experiment-log.md) یا README پروژه فنی.
5. به‌اشتراک‌گذاری بازخورد با تیم حاکمیت داده برای بهبود فرم‌ها، دسترسی‌ها و دستورالعمل‌های امنیتی.
